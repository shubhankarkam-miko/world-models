VLESA — Safety That Depends
on What You're Trying To Do

The same action — reaching for a knife, holding a cord — can be perfectly safe or genuinely dangerous. VLESA never scores an action in isolation: it first infers the wearer's goal from egocentric video, then asks a goal-conditioned safety filter whether this action, for this goal, crosses the line.

Stage 0 / 3 — idle
STAGE 1
👁️

Intent–Action Prediction

A frozen VLM (Llama-4-Scout) watches keyframes and jointly infers the latent goal ĝ and K candidate next actions, as scene-graph triplets.

STAGE 2
⚖️

Goal-Conditioned Q-Filter

A small VLM (Qwen3-VL-2B), fine-tuned via GRPO on EgoSafety, scores each candidate: Q(I,a,g) < 0 means safe under this specific goal.

STAGE 3
🚦

Constrained Decoding

Scores blend VLM rank with safety. If any candidate is safe, pick the best one; if none are, fall back to the least-unsafe and raise a danger alert.
