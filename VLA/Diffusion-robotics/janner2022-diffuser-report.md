# Planning with Diffusion for Flexible Behavior Synthesis

## TL;DR

Janner, Du, Tenenbaum, and Levine (ICML 2022) propose **Diffuser**, a denoising diffusion probabilistic model that operates directly on entire state-action trajectories rather than on single transitions. Instead of learning a one-step dynamics model and handing it to a separate trajectory optimizer, they fold planning into the generative model itself: sampling a trajectory from Diffuser *is* planning, and steering that sampling process (via classifier-style guidance or inpainting-style constraints) *is* how rewards and goals get incorporated. The claimed payoff is a planner that handles long horizons and sparse rewards gracefully, composes with new reward functions or constraints at test time without retraining, and generalizes to novel trajectories by stitching together locally-consistent subsequences it has seen before. It's the first paper to apply diffusion models to reinforcement learning / decision-making, borrowing techniques (classifier guidance, inpainting) wholesale from image diffusion and reinterpreting them as RL primitives.

## Architecture

**Trajectory representation.** A trajectory τ is represented as a 2D array with one column per planning timestep and states/actions stacked as rows within each column (Eq. 2). States and actions are predicted jointly — for prediction purposes, actions are just extra state dimensions. This is the same "trajectory as an image" trick that lets the rest of the architecture borrow directly from image diffusion.

**Diffuser (the denoising network).** Diffuser parameterizes the reverse diffusion process `p_θ(τ^{i-1} | τ^i)`, trained to predict the noise `ε_θ(τ^i, i)` added at diffusion step *i* (Fig. 2, Fig. A1). Structurally it is a **U-Net** lifted from image diffusion but with the 2D spatial convolutions replaced by **1D temporal convolutions** that slide along the planning-horizon axis — this gives equivariance along the horizon dimension while treating the state/action-feature dimension asymmetrically, matching the paper's design requirement that predictions be temporally local but not autoregressive. Concretely:
- 6 repeated residual blocks, each containing two temporal (1D) convolutions.
- Each convolution is followed by group normalization (Wu & He, 2018) and a Mish nonlinearity (Misra, 2019).
- A diffusion-timestep embedding is produced by a single fully-connected layer and injected additively into the activations after the first temporal convolution of each block.
- Because the network is fully convolutional along the horizon axis, the planning horizon T is *not* fixed by the architecture — it's set by the size of the input noise tensor `τ^N ~ N(0, I)`, so the same trained model can plan for variable-length horizons at test time (Fig. 3c).

**Guide function J_φ (reward/return predictor).** A separate network — architecturally the first half of the same U-Net plus a final linear layer — is trained to predict the cumulative return of a (possibly noisy) trajectory sample. The move of treating reward maximization as conditioning on trajectory "optimality" comes from the control-as-inference view of RL[^inference]; concretely, the gradient `∇J(μ)` is added to the mean of each reverse-diffusion step (Algorithm 1, line 8), which is exactly the mechanics of classifier-guided sampling[^guidance] (Dhariwal & Nichol, 2021) transplanted from "guide toward a class label" to "guide toward high return."

**Inpainting-based conditioning.** Hard constraints — the trajectory must start at the current observed state, or must reach a specified goal — are enforced the way image diffusion models handle inpainting: after each unperturbed reverse step, the known entries of τ (the constrained states/actions) are overwritten with their fixed values before continuing to the next denoising step. This is what lets Diffuser be reused across single-task, multi-task (resampled goals), and constraint-satisfaction settings without retraining the base model.

**The planning loop (Algorithm 1).** At each environment step: observe state *s*, initialize `τ^N ~ N(0, I)`, run N reverse-diffusion steps (each combining a Diffuser denoising step, a guide-gradient nudge, and an inpainting overwrite of the first state to *s*), execute the first action of the resulting plan `τ^0`, then repeat — i.e., a receding-horizon control loop where "replanning" means "resampling."

**Why the pieces fit together this way:** the paper's stated design goals are (1) non-autoregressive prediction of the whole trajectory at once (because planning requires conditioning on future goals, which breaks causal/autoregressive generation), (2) temporally-local computation within a single denoising step so that local consistency can be composed across steps into global coherence, and (3) horizon-equivariance so plan length is a sampling-time choice, not an architectural one. The U-Net-with-1D-convs, the guide function, and the inpainting mechanism are each a direct answer to one of these goals rather than independent design choices.

## Data

Diffuser is evaluated on three offline datasets, each testing a different capability:

- **Maze2D / Multi2D (D4RL, Fu et al. 2020):** offline trajectories of undirected maze navigation (a controller wandering to and from randomly selected locations), used to test long-horizon, sparse-reward planning. Multi2D is the same data but with the goal location randomized per episode at evaluation time to test multi-task generalization without retraining.
- **Block stacking:** 10,000 demonstration trajectories generated by the classical PDDLStream planner (Garrett et al., 2020) for a Kuka-arm block-stacking task, covering unconditional stacking, conditional (specified block order) stacking, and rearrangement. Rewards are sparse (1 on successful placement, 0 otherwise).
- **D4RL offline locomotion suite (Fu et al. 2020):** HalfCheetah, Hopper, and Walker2d, each with Medium-Expert, Medium, and Medium-Replay data quality tiers — testing recovery of a good controller from heterogeneous-quality offline data.

The paper does not discuss dataset biases, filtering, or preprocessing choices beyond citing the source benchmarks (D4RL, PDDLStream) — data curation is treated as inherited from those benchmarks rather than a contribution of this work.

## Training setup

- **Objective:** simplified DDPM noise-prediction loss, `L(θ) = E[‖ε − ε_θ(τ^i, i)‖²]`, with diffusion timestep `i ~ U{1,...,N}`.
- **Optimizer:** Adam, learning rate 4e-5, batch size 32, trained for 500k steps.
- **Diffusion steps N:** 20 for locomotion tasks, 100 for block stacking.
- **Noise schedule:** cosine schedule (Nichol & Dhariwal, 2021) for the reverse-process covariances.
- **Planning horizon T:** 32 for locomotion; 128 for block stacking; 128 / 265 / 384 for Maze2D-U-Maze / Medium / Large respectively.
- **Guide scale α:** 0.1 for all tasks except hopper-medium-expert (0.0001) — the paper notes guide scale needs to shrink substantially (e.g., to 0.001) when the planning horizon is shortened, indicating some sensitivity between horizon length and guidance strength.
- **Return predictor J_φ:** trained with discount factor γ = 0.997, though the paper reports control performance was insensitive to γ above 0.99.
- The paper notes predicting noise ε vs. predicting the clean trajectory τ⁰ directly made no substantial difference to control performance.
- **Warm-starting (Section 5.4):** rather than sampling fresh noise at every environment step, subsequent plans can be generated by partially re-noising the previous plan and running fewer reverse steps from there — trading off sample quality for a much smaller per-step planning budget.

## Evaluation & results

- **Long-horizon planning (Table 1, Maze2D/Multi2D):** Diffuser scores 113–132 (normalized, expert-relative) across all maze sizes in both single-task and multi-task settings, versus IQL's 13–59, and ground-truth-dynamics MPPI's single digits to low 40s. The multi-task setting requires no retraining for Diffuser (just changing the inpainted goal), whereas IQL needs a separately-adapted goal-conditioned variant and still drops in performance relative to its single-task version.
- **Test-time flexibility (Table 3, block stacking):** Diffuser scores 45–59 across unconditional stacking, conditional stacking, and rearrangement, versus CQL (0–24) and BCQ (0 on all three). The conditional tasks — which require composing two guide functions (final-state matching + contact constraint) — are where the gap is largest, since the model-free baselines have no analogous mechanism for composing constraints at test time.
- **Offline locomotion (Table 2, D4RL):** Diffuser averages 77.5 across the 9 HalfCheetah/Hopper/Walker2d × data-quality-tier combinations, comparable to but not dominant over specialized offline RL baselines (IQL 77.0, TT 78.9, CQL 77.6) — it beats other model-based approaches (MOReL 72.9, MBOP 47.8) and the return-conditioning baseline DT (74.7), but is not the best method on any individual task quality tier by a wide margin.
- **Fast planning (Fig. 7):** on Walker2d Medium-Expert, warm-started replanning can cut the number of diffusion steps roughly 10x with only minor performance degradation, provided each new plan is initialized from the previous plan rather than fresh noise.
- **Negative result reported by the authors:** substituting Diffuser as a learned dynamics model inside a conventional trajectory optimizer (MPPI) performed no better than random — the authors interpret this as evidence that Diffuser's effectiveness comes specifically from the coupling of modeling and planning into one sampling process, not from having learned an accurate one-step-ahead dynamics model that happens to be reusable elsewhere.

## Limitations (as stated by the authors)

- **Slow plan generation.** Individual plans are slow to produce because of the iterative (multi-step) denoising process; naively, a fresh plan must be regenerated from scratch at every control step under receding-horizon execution.
- **Not simply a better dynamics model.** The MPPI-with-Diffuser experiment performing no better than random is presented by the authors as showing a limitation in scope: Diffuser's gains are tied to its own coupled sampling/planning procedure and don't transfer to "drop-in dynamics model for any planner."
- **Locomotion performance is not best-in-class.** The authors explicitly note Diffuser performs worse than the best offline techniques designed specifically for single-task locomotion performance, positioning its value as flexibility/generality rather than raw single-task optimality.

## Critical read

**What's genuinely novel:** the framing, not any single mechanism. The paper's real contribution is recognizing that if you make trajectories the "images" that a diffusion model generates, then two well-established image-diffusion techniques — classifier guidance and inpainting — map almost verbatim onto two central RL problems (reward maximization and goal/constraint satisfaction), and that this mapping lets you dissolve the usual model/planner abstraction boundary entirely. The paper explicitly and correctly claims to be first to apply diffusion models to RL/decision-making (Section 6). That reframing is the paper's real intellectual contribution, more so than any individual architectural piece.

**What's an incremental transplant rather than a new invention:** essentially every individual component. The U-Net-with-temporal-convolutions is a direct, fairly mechanical port of the image-diffusion U-Net (2D convs → 1D convs). Classifier guidance for reward-steering is Dhariwal & Nichol's technique with `∇ log p(class)` relabeled as `∇ log p(optimality)`. Inpainting-as-constraint-satisfaction is literally the same inpainting mechanism used in image diffusion, applied to trajectory arrays instead of pixel arrays. None of this is a criticism exactly — the paper's thesis is precisely that these techniques transfer cleanly — but a reader should be clear that the novelty is "showing this transfer works and matters for RL," not "inventing new diffusion machinery."

**Open questions / places the results are softer than the headline suggests:**
- Planning budgets in Fig. 7 range from roughly 0.1–2 seconds *per replanning step*, even after warm-starting — this is slow relative to the sub-millisecond replanning typical of classical MPC/shooting methods, and the paper doesn't benchmark against those directly on wall-clock terms; the comparisons are score-vs-score against other *learned* offline RL methods, not against classical planners with ground-truth dynamics on speed.
- The guide scale α needing to shrink by orders of magnitude for shorter horizons or specific tasks (hopper-medium-expert) suggests the guidance mechanism is not particularly robust across settings and requires per-task tuning — a practical cost the headline "flexibility" framing understates.
- On the D4RL locomotion suite, Diffuser is "comparable to" rather than better than strong offline-RL baselines like IQL and TT — the paper is candid about this, but it does mean the strongest quantitative evidence for the method is in the sparse-reward/goal-conditioned/constraint-composition settings (Maze2D, block stacking) rather than in the setting (dense-reward locomotion) most commonly used to benchmark offline RL methods generally.
- The MPPI negative result is reported but not deeply investigated — it's a striking finding (a trained dynamics model that's actively harmful when handed to a different planner) that would benefit from more analysis of *why*, which the paper doesn't provide beyond the "coupled modeling and planning" hypothesis.

## Background: Borrowed Concepts

[^guidance]: **Classifier-guided diffusion** (Dhariwal & Nichol, 2021, *Diffusion Models Beat GANs on Image Synthesis*). To generate images conditioned on a class label *y*, this paper trains a separate classifier `p(y | x_t)` on noisy images at every noise level, then at each reverse-diffusion step nudges the model's predicted mean in the direction of `∇_x log p(y | x_t)` — the gradient of the classifier's log-probability with respect to the noisy sample — scaled by a tunable guidance strength. This steers samples toward whatever the classifier is confident is class *y*, at some cost to sample diversity, and was shown to meaningfully improve conditional image quality over just training a class-conditional diffusion model outright. The property Diffuser borrows wholesale is the *decoupling*: the base generative model is trained once, unconditionally, and a separate lightweight model supplies a gradient signal that steers sampling at test time — which is exactly how Diffuser lets one trained trajectory model be reused across different reward functions by swapping out J_φ, without retraining the diffusion model itself. (I have solid, specific knowledge of this paper.) [↩ back to Architecture](#architecture)

[^inference]: **Control-as-inference** (Levine, 2018, *Reinforcement Learning and Control as Probabilistic Inference: Tutorial and Review*). This tutorial reformulates optimal control as probabilistic inference by introducing, at each timestep, a binary "optimality" random variable `O_t` defined so that `p(O_t = 1 | s_t, a_t) ∝ exp(r(s_t, a_t))` — i.e., the probability that a timestep was "optimal" is exponentially proportional to its reward. Maximizing expected reward then becomes equivalent to inference: asking for the posterior over trajectories (or actions) conditioned on every timestep having been optimal, `p(τ | O_{1:T} = 1)`, rather than solving a separate optimization problem. This reframing is what lets Diffuser treat reward-maximization as just another distribution to sample from — the same diffusion-sampling machinery used for the base model applies unchanged, with the reward folded in as a conditioning event rather than an external optimization loop. I have general, non-expert familiarity with this framework — confident about this core construction (the optimality variable and its exponentiated-reward likelihood), but the tutorial itself develops considerably more machinery (soft value functions, connections to maximum-entropy RL algorithms) that isn't needed to follow Diffuser and that I'm not summarizing here.
