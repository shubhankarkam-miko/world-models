# Value-Guided Diffusion for Trajectory Planning: A Joint Read of Two Papers

**Paper A:** *Generative Trajectory Planning in Dynamic Environments: A Joint Diffusion and Reinforcement Learning Framework* — Anonymous, under review, ICLR 2026.
**Paper B:** *Generative Multi-Robot Motion Planning via Diffusion Modeling with Multi-Agent Reinforcement Learning Guidance* — Lee, Mutta, Ko (Arizona State University), arXiv:2606.00933v1 [cs.RO], May 2026.

## TL;DR

Both papers pair a diffusion model that proposes trajectories with a reinforcement-learning signal that decides what to do with them, but they are not solving the same problem and do not implement "guidance" the same way. Paper A is a **single-agent** UAV planner: diffusion generates a small set of candidate sub-paths at each receding-horizon step, and a SAC policy picks (or mixes) among them — RL never touches the diffusion sampling process itself. Paper B is genuinely **multi-agent**: a shared single-agent diffusion model samples trajectories independently per robot, and a centralized MARL-trained value function's *gradient* is injected directly into each reverse-diffusion step, steering the denoising trajectory toward coordinated behavior — the textbook classifier-guidance mechanism, extended to multi-robot coordination. Paper A reports strong simulated results but its appendix baseline comparisons contain internally inconsistent numbers worth treating with skepticism. Paper B's core result (inter-agent interference reduced from 55.4% to 41.8%) is a modest, plausible, well-scoped improvement with an honestly reported trade-off (obstacle clearance gets worse).

## Architecture

| | Paper A | Paper B |
|---|---|---|
| Diffusion role | Proposes K candidate **sub-paths** per receding-horizon step, each tagged with uncertainty, collision probability, length | Proposes a **full trajectory** (176-step horizon) per agent, conditioned on start/goal, trained once on single-agent data |
| Denoiser | Transformer (d_model 256, 4 heads, 6 layers), v-prediction, classifier-free guidance (CFG scale 2–3) | Temporal U-Net, noise-prediction (ε-parameterization), 256 diffusion steps |
| RL role | SAC (max-entropy) outputs a K-dim score vector → softmax/argmax over the candidate set → **discrete selection**, not gradient steering | MAPPO (centralized training, decentralized execution) trains a value function \(V_\phi(X)\) over the **joint** state of all agents |
| Coupling mechanism | Selection: `a* = argmax_a Q(s,a)` over candidates already fully generated by diffusion | Guidance: gradient of \(V_\phi\) added directly to the denoising mean at every reverse-diffusion step — \(\tilde p_\theta(X) \propto p_\theta(X)\exp(\lambda V_\phi(X))\) |
| Safety layer | A separate "Shield" applies smoothing/lateral offset on detected near-misses, independent of the RL/diffusion loop | None — coordination emerges purely from the guidance gradient; no explicit collision shield |
| Scope | 1 UAV, static + dynamic obstacles | 4 ground robots (Leatherback), static maze walls, no dynamic obstacles |

The naming convergence ("value-guided") is doing more work than the mechanisms actually share. Paper A's RL component operates entirely **after** generation — it never perturbs the trajectory the diffusion model already produced, it just chooses among finished options. Paper B's guidance operates **during** generation, at every one of the 256 denoising steps, which is what makes "guidance" the right word for it in the diffusion literature's sense (Dhariwal & Nichol classifier guidance; Janner et al.'s Diffuser inference-time guidance). If you're using either paper as a template for building "value-guided diffusion," Paper B is the one that actually demonstrates the pattern; Paper A demonstrates diffusion-as-candidate-generator plus a bandit-style selector.

## Data

**Paper A:** No offline dataset is described. Training appears to be online/on-policy: the diffusion model is trained jointly with SAC using a "denoising + smoothness + clearance" loss inside the same training loop (Algorithm 1). The paper never specifies where the initial x₀ sub-paths used to train the denoiser come from before the joint loop has produced any good trajectories — this is a gap, not fully specified.

**Paper B:** Explicit two-stage data pipeline. (1) A single-agent PPO policy is trained in a 30×30m maze to collect 2,800 navigation trajectories; the diffusion model is trained offline on this fixed dataset. (2) A *separate* MAPPO value function is trained in an open-space environment (no maze walls) with 4 robots holding relative formation offsets from a group centroid. These two training environments never see each other — the critic has no exposure to the obstacle geometry the diffusion model was trained on. This split is explicitly named by the authors as the source of their reported trade-off (see Limitations).

## Training setup

**Paper A** (Appendix, Tables 2–3): diffusion — 500 epochs, batch 64, 1000 diffusion steps, K=12 candidates/step, DDIM sampling (50 steps offline / 10 steps online, η=0, CFG 2.0–3.0). SAC — 1000 episodes, γ=0.99, target update τ=0.005, replay buffer 400k, actor/critic MLP [256,256], auto-tuned entropy temperature. Reward is a 7-term weighted sum (goal, collision, progress, distance, uncertainty, curvature, clearance) with hand-set weights (e.g. r_collision=500, r_goal=1000).

**Paper B**: PPO single-agent policy trained with the standard clipped surrogate objective; diffusion trained for 40 epochs (loss curve drops from ~37 to near-0, Fig. 4) — a short, offline supervised run, not interleaved with RL. MAPPO trained separately (centralized critic, decentralized actors) — no hyperparameter table is given for this stage. Critically: **guidance is applied only at inference time**. No gradients flow between the value function and the diffusion model's weights; the diffusion model is never retrained or fine-tuned with the coordination signal. This is a deliberate design choice the paper is explicit about (compositionality without retraining), not an oversight.

## Evaluation & results

**Paper A**, Table 1 (main body): segment=32 gets 1.00 success / 0.00 collision / 392.9 energy / 59.2 clearance — the best row in the table, beating segment=16, full-horizon diffusion-RL, and all heuristic/classical/pure-diffusion baselines. The appendix extends this with six more diffusion-planner baselines (Diffuser, MPD, MMD, SMD, MCTD, Stitch) — see Critical Read below, these numbers don't hold up to a consistency check.

**Paper B**, Table 1: a single headline metric, inter-agent interference rate \(R_{agent}\), averaged over 10 runs: baseline diffusion-only = 55.4%, MARL-guided = 41.8% (Δ = −13.6 points). Auxiliary metric \(R_{obs}\) (obstacle proximity) goes the wrong way: 4.6% → 7.6%, an explicit and honestly reported trade-off. The qualitative section (6.2) reports different single-run numbers for the illustrated trajectories (59.7% → 33.5%) — a "representative example," not the averaged result, but the paper doesn't flag the gap between the two figures, which is easy to misread as the headline result.

## Limitations (as stated by the authors)

**Paper A** (drawn from the Conclusion / Future Work, since there is no dedicated Limitations section): scaling to higher-dimensional systems, integrating semantic/vision-language information into candidate generation, no theoretical safety/stability guarantees yet, computational overhead of diffusion sampling, no real-hardware validation, no standardized large-scale benchmark suite used.

**Paper B** (Discussion, §6.3, the more candid of the two): the MARL critic is trained in an *obstacle-free* environment and has no encoding of maze geometry, which the authors identify as the direct cause of the increased wall-proximity trade-off. They also note the guided trajectories show oscillation artifacts from the per-step gradient corrections, explicitly framed as a side effect rather than a benefit. Future work: scaling beyond 4 agents (untested), training the critic in an obstacle-aware setting, and physical hardware validation.

## Critical read

**1. The shared "theme" is real but narrower than advertised.** Only Paper B does multi-agent motion planning. Paper A is single-agent trajectory selection under dynamic obstacles — a legitimate and separately interesting problem, but not the multi-robot coordination problem the combined framing implies. If the goal is to survey "value-guided diffusion for multi-agent planning" as a subfield, Paper A is background/contrast material, not a direct example.

**2. "Guidance" means two different things here, and only one of them is guidance in the standard diffusion sense.** Paper A: candidate generation → discrete post-hoc selection (closer to rejection sampling / a learned re-ranker). Paper B: gradient of a value function added to the denoising mean at every reverse step (Eq. 17–18), which is exactly the classifier-guidance formulation from Dhariwal & Nichol (2021) and the inference-time guidance Janner et al. (2022) demonstrated for Diffuser — extended here from single-objective scalar rewards to a MARL-trained joint critic. That extension (compositional guidance from an independently-trained multi-agent critic, with no retraining of the generative model) is the one genuinely novel technical contribution across both papers.

**3. Paper A's appendix baseline table doesn't hold together.** The main-body Table 1 reports the "pure diffusion model" baseline at 0.27 success, 0.01 collision, 406.1 energy. The appendix table (page 15) then lists Diffuser, MPD, MMD, SMD, and Stitch all at *exactly* 0.00 success / 1.00 collision, with energy costs 6–12× higher than any other row (2417–4918), and MCTD at 0.00 success / 0.00 collision / 1.00 timeout with a clearance value (26.07) inconsistent with everything around it. Well-known planners producing perfectly uniform total-failure numbers, in a table that isn't referenced or discussed anywhere in the main text, is a pattern worth treating as unverified rather than as evidence these methods were faithfully reproduced. This doesn't necessarily undermine the paper's core comparison (against heuristic, classical, and its own diffusion-only ablation, which are internally consistent) — but the extended baseline table shouldn't be cited as-is.

**4. Paper A shows signs of light editing / possible generation artifacts.** A garbled equation reference ("recover a cleaner x + t − 1"), a paragraph in §4.1 that duplicates the Figure 1 caption almost verbatim, and some run-on phrasing suggest the draft hasn't had a careful editing pass. Combine this with point 3, and I'd read Paper A's quantitative claims as directionally plausible but not fully verified — appropriate treatment for an anonymous paper still under double-blind review.

**5. Paper B is more careful about what it claims**, and is correspondingly more useful as a template: it reports its own trade-off rather than only the favorable metric, distinguishes averaged results from a single illustrative run (even if not clearly enough), and is explicit that guidance happens only at inference with no retraining — a real, checkable, and modest claim rather than a sweeping one.

**6. Novelty, net:** Paper A's contribution is largely an engineering combination — diffusion for diverse proposals, SAC for selection, a hand-tuned multi-term reward, a smoothing shield — with a straightforward complexity argument (\(O(M \cdot T_{sub} \cdot H)\) vs. \(O(M \cdot T \cdot H)\)) rather than a new algorithm. Paper B's contribution is narrower but cleaner: it's the first (per its own literature review, plausibly correct) application of classifier-style value guidance to *decentralized* multi-robot diffusion planning using a *separately trained* MARL critic, with no joint retraining required. That's a real, specific claim, and the experiment (even if small-scale, 4 agents, single maze) is designed to test exactly that claim.

**7. Both papers are sim-only.** UAV simulation (unspecified engine) for A, Isaac Sim for B. Neither has hardware validation; both name this as future work. Given that Paper A's simulator and energy model aren't described in much detail, its absolute numbers (e.g., "energy consumption 348.8") aren't independently interpretable — they're only meaningful as within-paper relative comparisons.
